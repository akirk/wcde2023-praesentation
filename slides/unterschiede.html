Vergleich zu RSS

  • RSS: Pull "Ziehen" → regelmäßiges Abrufen, selten Neues
    • Für Clients gemacht, die auch mal offline sind
    • Auf Texte fokussiert, statische Datei reicht
    • XML-basiert, dadurch erweiterbar (Podcasts)
  • ActivityPub: Push "Schieben" → Neue Beiträge werden zugestellt
    • Server wird benötigt für direkte Zustellung
    • Effizienter, da nur Aufrufe bei Neuigkeiten
    • Aktionen werden abgebildet: Create Update Delete
    • JSON
